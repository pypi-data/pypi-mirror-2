 Ein paar Notizen für Entwickler

    virtualenv
    ~~~~~~~~~~
 
 run only the script:
    create_test_env.sh
 
 the hard way:
    wget http://svn.pylucid.net/pylucid/branches/0.9/scripts/pylucid-boot.py
    python pylucid-boot.py pylucid-env
    cd pylucid-env
    source bin/activate
    pip install --requirement src/pylucid/requirements/external_apps.txt

    Test SQlite Datei
    ~~~~~~~~~~~~~~~~~
 SQLite Datei pylucid_project/test.db3 enthält gefüllte Tabellen einer PyLucid v0.8 installation.
 Existierende User:
    Name....: test
    Passwort: 12345678
 
 Es muß also die neuen v0.9 Tabellen erst angelegt werden: 
    ./pylucid_project/manage.py syncdb
    
 Die existierende test.db3 SQLite Datei sollte nicht comittet werden! Damit man immer schnell 
 zurück kann...
 
 
    Test Sever
    ~~~~~~~~~~
 Nach syncdb kann man den test Server starten, mit:
    ./pylucid_project/manage.py syncdb
